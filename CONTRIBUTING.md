# Contributing

Thank you for your interest in contributing to this PostgreSQL Integrity Project.

## How to Contribute

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

## Guidelines

- Keep SQL scripts idempotent
- Follow PostgreSQL best practices
- Document any new scripts added
- Update README when adding new features
- Ensure install.sh continues to work end-to-end

## Bug Reports

Please open an issue describing:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details
